nio-buffer

Buffer扮演的角色

可写入数据也可从中读取数据的内存

Buffer的三个基本概念： capacity limit position

  • capacity : 缓存的容量，当存入的数据超过容量后，只有清除缓存后才能继续往里写数据
  • position : 写模式下写到的数据位置，初始为0，最大为capacity-1;写模式下，重置为0，向前移动 到下一个可读的位置
  • limit : 写模式下，等于capacity;读模式下，等于写模式下的position【你能读到之前写入的所有数据的上限】

Buffer的分类

除Boolean外的其他其他数据类型均可存入Buffer中
ByteBuffer ShortBuffer IntBuffer LongBuffer CharBuffer DoubleBuffer FloatBuffer

  • MapperByteBuffer【有些特别，以后章节专门介绍】

Buffer的内存分配：利用每个Buffer类的allocate方法

ByteBuffer buf = ByteBuffer.allocate(48); //分配48个字节的ByteBuffer

Buffer写模式

  • 从Channel中读到缓存中： int bytesRead = inChannel.read(buf);
  • 直接put: buf.put(127);

Buffer读模式

  • 从缓存读到通道中： int bytesRead = inChannel.write(buf);
  • 直接拿 ： buf.get();

Buffer的其他方法：

  • 切换到读模式： flip()
  • 重新读取数据： rewind()
  • 以上两种方法有类似的效果，但是切换的过程有所不同，前者是从写切换到读，后者是一直处于读模式

  • 清空缓存，数据读完，准备下一次写入：clear(),compact()

    • clear:清空所有数据，包括未读的。【覆盖原有的数据】
    • compact: 标定未读数据的位置，写数据从最后一个未读数据开始。

  • mark: 重置position

  • reset: 恢复重置的position
