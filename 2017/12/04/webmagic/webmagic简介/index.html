webmagic01-概览

webmagic四个基本组件：

Downloader、PageProcessor、Scheduler、Pipeline

  • 核心包:webmagic-core 扩展包： webmagic-extension
  • 开发模式：注解开发，扩展了XPath语法的Xsoup

Downloader

负责从互联网下载页面，以便后续处理，默认使用Apache HttpClient作为下载工具。

PageProcessor

  • 负责页面的解析，抽取有用信息，以及发现新链接。
  • 解析工具：Jsoup，以及基于其开发的解析XPath的Xsoup
  • 在这四个组件中，PageProcessor对于每个站点每个页面都不一样，是需要使用者定制的部分

Scheduler

负责管理带抓取的URL，以及去重

  • 提供的内存队列实现管理
  • 用集合去重
  • 也支持Redis进行分布式管理

Pipeline

负责结果的处理。【1 输出到控制台 2 保存到文件】

  • 计算
  • 持久化到文件
  • 数据库

数据流转对象

1 Request

对URL的封装，PageController和Downloader的交互载体。

2 Page

代表Downloader下载的一个页面，可能是HTML，JSON,或其他文本格式。

3 ResultItems

ResultItems相当于一个Map，它保存PageProcessor处理的结果，供Pipeline使用。
它的API与Map很类似，值得注意的是它有一个字段skip，若设置为true，则不应被Pipeline处理。

Spider–控制爬虫运转的引擎

1、pider是WebMagic内部流程的核心。

2、Downloader、PageProcessor、Scheduler、Pipeline都是Spider的一个属性

3、以上四个属性的不同配置可以实现不同的功能，但开发者并不需要关心太多，因为webmagic已经提供了默认实现。
【一般来说，编写一个爬虫，PageProcessor是需要编写的部分，而Spider则是创建和控制爬虫的入口】

以下是一个简单的例子：

public static void main(String[] args) {
    Spider.create(new GithubRepoPageProcessor())
            //从https://github.com/code4craft开始抓    
            .addUrl("https://github.com/code4craft")
            //设置Scheduler，使用Redis来管理URL队列
            .setScheduler(new RedisScheduler("localhost"))
            //设置Pipeline，将结果以json方式保存到文件
            .addPipeline(new JsonFilePipeline("D:\\data\\webmagic"))
            //开启5个线程同时执行
            .thread(5)
            //启动爬虫
            .run();
}
